# MicroGrid Pro - Two-Factor Authentication (2FA)

## Overview

Two-Factor Authentication (2FA) provides an extra layer of security by requiring users to provide both a password and a time-based one-time password (TOTP) generated by authenticator apps like Google Authenticator, Microsoft Authenticator, or Authy.

## Features

✅ **TOTP (Time-based One-Time Password)**
- Compatible with Google Authenticator, Microsoft Authenticator, Authy, FreeOTP
- 6-digit codes that change every 30 seconds
- Configurable time window for clock skew tolerance

✅ **Backup Codes**
- 8 single-use backup codes for account recovery
- Codes are bcrypt-hashed for storage security
- Automatically consumed after use

✅ **Audit Trail**
- 2FA verification log table for monitoring
- Track success/failure, IP address, user agent
- Supports compliance and security investigations

✅ **Secure Implementation**
- Base32 encoding for secrets (RFC 4648)
- HMAC-SHA1 for TOTP generation
- Bcrypt for backup code hashing
- Rate limiting on API endpoints

## Installation

### Step 1: Run Database Migration

Execute the migration to add 2FA fields to users table:

```bash
mysql -u root -p microgrid_platform < database/migrations/20260315_add_2fa_support.sql
```

Or via PHP:
```php
php database/restore.php database/migrations/20260315_add_2fa_support.sql
```

This creates:
- `users.two_fa_enabled` - Boolean flag
- `users.two_fa_secret` - Base32-encoded TOTP secret
- `users.two_fa_backup_codes` - JSON-encoded hashed backup codes
- `users.two_fa_enabled_at` - Timestamp when 2FA was enabled
- `two_fa_verification_log` - Audit trail table

### Step 2: Include the Libraries

The 2FA system is automatically loaded when you include `config/database.php`:

```php
require_once 'config/database.php'; // Loads everything including 2FA
```

Or include manually:
```php
require_once 'includes/twofactor.php';
require_once 'includes/login_2fa_helper.php';
```

## API Endpoints

All endpoints at `/api/twofactor.php`

### 1. Enable 2FA (Start Setup)

**Endpoint:** `POST /api/twofactor.php?action=enable`

**Authentication:** Session required (user must be logged in)

**Response:**
```json
{
  "success": true,
  "message": "Scan QR code with authenticator app",
  "qr_url": "otpauth://totp/MicroGrid%20Pro%20(user@example.com)?secret=...",
  "secret": "JBSWY3DPEBLW64TMMQ======",
  "backup_codes_preview": {
    "count": 8,
    "example": "A1B2C3D4"
  }
}
```

**Usage:**
```javascript
// Step 1: Get QR code URL
const response = await fetch('/api/twofactor.php?action=enable', {
    method: 'POST'
});

const data = await response.json();
// Display data.qr_url as QR code to user
// User scans with authenticator app
```

### 2. Verify 2FA Setup

**Endpoint:** `POST /api/twofactor.php?action=verify`

**Parameters:**
- `code` (string, 6 digits) - Code from authenticator app

**Response:**
```json
{
  "success": true,
  "message": "2FA setup complete",
  "backup_codes": [
    "A1B2C3D4",
    "E5F6G7H8",
    "...6 more"
  ]
}
```

**Usage:**
```javascript
// Step 2: User enters 6-digit code and clicks Verify
const code = document.getElementById('code').value; // "123456"

const response = await fetch('/api/twofactor.php?action=verify', {
    method: 'POST',
    body: new URLSearchParams({ code })
});

const data = await response.json();
if (data.success) {
    // Display backup codes to user
    // User must save these codes!
    console.log('Backup codes:', data.backup_codes);
}
```

### 3. Disable 2FA

**Endpoint:** `POST /api/twofactor.php?action=disable`

**Parameters:**
- `password` (string) - User's password for verification

**Response:**
```json
{
  "success": true,
  "message": "2FA has been disabled"
}
```

**Usage:**
```javascript
const password = prompt('Enter your password to disable 2FA:');
const response = await fetch('/api/twofactor.php?action=disable', {
    method: 'POST',
    body: new URLSearchParams({ password })
});
```

### 4. Generate New Backup Codes

**Endpoint:** `POST /api/twofactor.php?action=backup-codes`

**Parameters:**
- `code` (string, 6 digits) - Current TOTP code for verification

**Response:**
```json
{
  "success": true,
  "message": "New backup codes generated",
  "backup_codes": ["...", "..."]
}
```

**Usage:**
```javascript
const code = document.getElementById('twoFACode').value;
const response = await fetch('/api/twofactor.php?action=backup-codes', {
    method: 'POST',
    body: new URLSearchParams({ code })
});
```

### 5. Verify Login (During Authentication)

**Endpoint:** `POST /api/twofactor.php?action=verify-login`

**Parameters:**
- `user_id` (int) - User ID from initial password authentication
- `code` (string, 6 digits or 8 characters) - TOTP or backup code

**Response (Success):**
```json
{
  "success": true,
  "message": "2FA verification successful",
  "code_type": "totp"
}
```

**Response (Failure):**
```json
{
  "error": "Invalid authentication code"
}
```

## Login Integration Example

### Basic Login Flow with 2FA

```php
<?php
require_once 'config/database.php';
require_once 'includes/login_2fa_helper.php';

if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    try {
        // Step 1: Verify username and password
        $user = authenticate_by_password(
            $_POST['username'],
            $_POST['password']
        );

        // Step 2: Check if 2FA is enabled
        if ($user['two_fa_enabled']) {
            // 2FA required - this function will send 403 response
            handle_2fa_requirement($user);
            // Code below doesn't execute
        }

        // Step 3: 2FA not enabled - create session
        $_SESSION['user_id'] = $user['id'];
        $_SESSION['username'] = $user['username'];
        header('Location: /dashboard.php');

    } catch (Exception $e) {
        $error = $e->getMessage();
    }
}

// If we have an active 2FA challenge, show 2FA form instead of login form
$show2FAForm = !empty($_SESSION['2fa_challenge']);
?>

<!-- Login Form -->
<?php if (!$show2FAForm): ?>
    <form method="POST">
        <input type="text" name="username" required>
        <input type="password" name="password" required>
        <button type="submit">Login</button>
    </form>
<?php else: ?>
    <!-- 2FA Verification Form -->
    <form method="POST" action="/login.php?action=verify-2fa">
        <input type="hidden" name="user_id" value="<?= $_SESSION['2fa_challenge']['user_id'] ?>">
        <input type="text" name="code" maxlength="8" placeholder="123456 or A1B2C3D4" required>
        <button type="submit">Verify</button>
    </form>
<?php endif; ?>
```

## Using the TwoFactorAuth Class

### Generate a New Secret

```php
$secret = TwoFactorAuth::generateSecret();
// Returns: "JBSWY3DPEBLW64TMMQ======"
```

### Verify a TOTP Code

```php
$isValid = TwoFactorAuth::verifyToken(
    'JBSWY3DPEBLW64TMMQ======',
    '123456'
);
// Returns: true or false
```

### Generate QR Code URL

```php
$url = TwoFactorAuth::getQRCodeUrl(
    'JBSWY3DPEBLW64TMMQ======',
    'user@example.com',
    'MyApp'
);
// Returns: "otpauth://totp/...?secret=...&issuer=MyApp"
// Display this as QR code using qrserver.com or similar
```

### Generate Backup Codes

```php
$codes = TwoFactorAuth::generateBackupCodes(8);
// Returns: ["A1B2C3D4", "E5F6G7H8", ...]
```

### Hash Backup Codes for Storage

```php
$hashed = TwoFactorAuth::hashBackupCodes($codes);
// Returns: JSON string with bcrypt-hashed codes
// Store in database: users.two_fa_backup_codes
```

## Security Considerations

### Best Practices

1. **Backup Codes**
   - Users MUST save backup codes immediately after 2FA setup
   - Backup codes cannot be recovered if lost
   - Each backup code can only be used once
   - Encourage users to store codes in a password manager

2. **Secret Storage**
   - Secrets are stored in plaintext in database (necessary for TOTP verification)
   - Use encrypted database backups
   - Restrict database access to authorized personnel only

3. **Time Synchronization**
   - Server time must be within 1-2 minutes of user's device time
   - Authenticator apps handle this automatically
   - If time is too far off, codes will be invalid

4. **Rate Limiting**
   - API endpoints are rate-limited (30 requests/minute)
   - Login attempts are logged to audit trail
   - Monitor for brute force attempts

5. **HTTPS Required**
   - Always enforce HTTPS for 2FA endpoints
   - QR codes and secrets should never be transmitted over HTTP
   - Configured in `.htaccess` to redirect HTTP to HTTPS

### Monitoring

Check the 2FA verification log for security issues:

```sql
-- View all 2FA verification attempts
SELECT * FROM two_fa_verification_log 
ORDER BY verification_time DESC 
LIMIT 100;

-- Find failed attempts
SELECT user_id, COUNT(*) as failures, MAX(verification_time) as last_attempt
FROM two_fa_verification_log 
WHERE success = 0 
GROUP BY user_id 
HAVING failures > 10;

-- Track by IP address
SELECT ip_address, user_id, COUNT(*) as attempts
FROM two_fa_verification_log 
WHERE verification_time > DATE_SUB(NOW(), INTERVAL 1 HOUR)
GROUP BY ip_address, user_id
HAVING attempts > 20;
```

## Troubleshooting

### "Invalid code even though I'm entering correct code"
- **Cause:** Server and device time are out of sync
- **Solution:** Set device time to correct time using NTP
- **Note:** Time can drift up to 90 seconds (3 × 30-second windows)

### "Backup codes not working"
- **Cause:** Backup code has already been used
- **Solution:** Generate new backup codes after using one
- **Cause:** Incorrect format (should be 8 characters)
- **Solution:** Ensure user is entering uppercase letters

### "Can't scan QR code"
- **Cause:** QR code image not loading
- **Solution:** Check that qrserver.com is accessible from user's network
- **Alternative:** Manually enter the secret key instead of scanning QR

### "User locked out after 5 failed attempts"
- **Solution:** User can wait a few minutes for rate limit to reset
- **Solution:** Use browser developer tools to clear rate limiter file if stuck
- **Admin help:** Disable 2FA temporarily, then have user set it up again

## Migration from Old System

If migrating from an existing login system:

1. All users start with `two_fa_enabled = 0`
2. Users can opt-in to 2FA via profile settings
3. No forced upgrade - migration is gradual
4. Existing sessions continue to work normally

## Testing

### Unit Tests for TwoFactorAuth

```php
// Test basic TOTP functionality
$secret = TwoFactorAuth::generateSecret();
$code = TwoFactorAuth::verifyToken($secret, '000000'); // Will fail - testing

// Test QR code URL generation
$qrUrl = TwoFactorAuth::getQRCodeUrl($secret, 'test@example.com');
assert(strpos($qrUrl, 'otpauth://totp/') === 0);

// Test backup codes
$codes = TwoFactorAuth::generateBackupCodes();
assert(count($codes) === 8);
assert(strlen($codes[0]) === 8);
```

### Integration Test

```php
// Simulate full 2FA flow
$secret = TwoFactorAuth::generateSecret();

// User enters wrong code
assert(!TwoFactorAuth::verifyToken($secret, '000000'));

// Generate backup codes
$codes = TwoFactorAuth::generateBackupCodes();
$hashed = TwoFactorAuth::hashBackupCodes($codes);

// User uses backup code
assert(TwoFactorAuth::verifyBackupCode($codes[0], json_decode($hashed, true)));

// Code is consumed
$newHashed = TwoFactorAuth::consumeBackupCode($codes[0], $hashed);
assert($newHashed !== $hashed);
assert(!TwoFactorAuth::verifyBackupCode($codes[0], json_decode($newHashed, true)));
```
